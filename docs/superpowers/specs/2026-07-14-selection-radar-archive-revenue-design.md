# 墨西哥选品雷达：名称、日期筛选与美元月销售额设计

日期：2026-07-14
状态：已获方向确认，待实施

## 1. 背景与问题

当前公开预览域名 `mexico-product-radar.oumursgpt001.chatgpt.site` 在用户网络中被 Cloudflare 拦截并返回 403。这不是中文编码乱码。GitHub Pages 主站可正常访问，因此公开分享与飞书推送不应再依赖容易被拦截的预览域名。

同时，现有页面需要完成三项产品化升级：

1. 用户可见名称从“新品”统一调整为“选品”。
2. 历史记录增加按简报日期筛选，并与现有关键词、平台、结论筛选联动。
3. 每个推荐产品增加“单链接月销售额预估”，统一换算并展示为美元，同时明确数据来源、计算方法、周期和可信度。

## 2. 目标

- GitHub Pages 成为公开访问、飞书分享和历史翻查的主链接。
- 页面、元数据、简报标题与飞书卡片统一使用“墨西哥选品雷达 / 选品简报”。
- 历史记录支持精确日期筛选，且可与其他条件组合。
- 月销售额只在证据链完整时给出美元区间，不制造伪精确数字。
- 旧简报数据无需一次性迁移即可继续正常渲染。

## 3. 非目标

- 不更改 GitHub 仓库名、Pages 路径或已有简报 URL。
- 不把评论数、排名、Google Trends 或搜索结果热度直接换算为销量或销售额。
- 不承诺免费调用 FastMoss 等需要授权、付费账户或受限接口的数据。
- 不因备用预览站发布失败而阻断 GitHub Pages 或飞书推送。

## 4. 用户可见名称

以下用户可见文案统一替换：

- “墨西哥新品雷达”改为“墨西哥选品雷达”。
- “新品简报”改为“选品简报”。
- 自动任务名称与飞书卡片标题中的“新品”改为“选品”。

仓库名、目录名、历史 URL 等技术标识保持不变，避免破坏现有链接。

## 5. 发布与分享策略

### 5.1 主站

GitHub Pages 是唯一必须成功的公开发布目标：

`https://grappen0728.github.io/mexico-product-radar/`

飞书消息只发送 GitHub Pages 首页和当日简报链接。

### 5.2 备用预览

Sites 预览保留为尽力而为的备用发布渠道。发布失败或访问被 Cloudflare 拦截时：

- 记录警告；
- 不阻断飞书推送；
- 不把该链接作为用户主入口；
- 不将 403 页面描述为乱码。

## 6. 历史记录日期筛选

### 6.1 交互

历史记录筛选区新增“简报日期”下拉框：

- 默认项为“全部日期”。
- 选项来自现有简报的报告日期，去重后按日期倒序排列。
- 选择某日后，只显示该日简报。
- 日期条件与关键词、平台、结论条件同时生效。
- 清空筛选时恢复“全部日期”。

### 6.2 数据与静态站

- React 过滤类型增加可选字段 `date?: string`。
- 静态历史卡片增加 `data-date="YYYY-MM-DD"`。
- 静态脚本读取日期控件并参与组合过滤。
- 筛选布局在桌面端容纳四个控件，在窄屏下保持可用。

## 7. 单链接月销售额预估

### 7.1 展示原则

每个产品都显示“单链接月销售额预估”模块，但只有证据链完整时才显示金额区间。金额统一为美元，格式示例：

`US$8,200–US$11,600 / 月`

同时展示：

- 对应的单一商品链接；
- 估算周期；
- 月销量区间；
- 商品单价及原始币种；
- 汇率、汇率日期与汇率来源；
- 估算方法；
- 可信度：高 / 中 / 低；
- 支撑该估算的来源链接。

若证据不足，则显示“暂无可靠估算”，并说明缺失项，例如“缺少可核查的近 30 天销量”或“缺少同日可靠汇率”。

### 7.2 数据结构

推荐记录增加可选对象 `monthlyRevenueEstimate`：

```ts
type MonthlyRevenueEstimate = {
  status: "available" | "unavailable";
  sourceProductUrl: string | null;
  period: string | null;
  estimatedMonthlyUnitsMin: number | null;
  estimatedMonthlyUnitsMax: number | null;
  unitPrice: number | null;
  unitPriceCurrency: string | null;
  mxnPerUsd: number | null;
  fxCapturedAt: string | null;
  fxSourceUrl: string | null;
  fxPublisher: string | null;
  method: string | null;
  confidence: "high" | "medium" | "low" | null;
  sourceUrls: string[];
  unavailableReason: string | null;
};
```

美元销售额优先由渲染前的计算函数产生，不要求内容作者重复手填最终金额，从而避免输入值与公式结果不一致。

### 7.3 计算口径

墨西哥比索商品的统一公式：

```text
月销售额下限（MXN） = 月销量下限 × 商品单价（MXN）
月销售额上限（MXN） = 月销量上限 × 商品单价（MXN）
月销售额（USD） = 月销售额（MXN） ÷ 每 1 美元对应的 MXN 汇率
```

若商品原始价格已经是美元，则不重复换算，但仍记录币种与核查日期。其他币种只有在存在可靠的对应美元汇率时才可展示。

### 7.4 可接受的销量依据

可用于估算的销量输入仅包括：

1. 平台官方页面或获授权数据源直接给出的近 30 天 / 自然月销量；
2. 累计销量配合可核查的上架时间或明确观察窗口推导出的月销量区间，此时必须下调可信度并公开方法。

以下指标只能作为趋势旁证，不能单独换算销售额：

- 评论数或评分数；
- 类目排名；
- Google Trends；
- 搜索结果数量；
- 视频播放、点赞或互动量；
- “热卖”“爆款”等平台标签。

### 7.5 汇率依据

- 使用报告核查日可访问的公开权威汇率来源，优先央行或官方金融数据源。
- 记录发布方、汇率值、日期和原始链接。
- 汇率日期原则上与商品价格及销量核查日期一致；若非交易日只能使用最近一个公开工作日，并明确日期。
- 无法核查汇率时不展示美元金额。

### 7.6 可信度规则

- 高：销量为平台或授权源的明确 30 天数据，价格与汇率均可核查。
- 中：销量为明确时间窗口内的可靠推导，所有输入均可核查。
- 低：输入仍可审计，但观察窗口短、区间宽或数据存在合理不确定性。
- 任何关键输入缺失：状态为 `unavailable`，不输出金额。

## 8. 数据流与自动简报

每日任务按以下顺序运行：

1. 为 TikTok Shop Mexico、Temu Mexico、Mercado Libre Mexico 各选一个产品。
2. 核查商品链接、价格、销量依据、媒体与来源可访问性。
3. 尝试生成月销量区间与美元月销售额；证据不足时保留缺失说明。
4. 生成当日选品简报和历史索引。
5. 运行结构、公式、链接和页面测试。
6. 发布 GitHub Pages；只有该步骤成功才发送飞书消息。
7. 尝试更新备用 Sites；失败只记录警告。

## 9. 兼容性

- `monthlyRevenueEstimate` 为可选字段，旧简报未提供时按“暂无可靠估算”处理。
- 旧 URL 与历史归档路径保持不变。
- 日期筛选读取报告日期，不依赖文件创建时间。

## 10. 验收标准

### 10.1 文案与访问

- 公开页面、浏览器标题、描述、简报标题与飞书卡片不再把产品称为“新品”。
- GitHub Pages 首页和当日简报可直接访问。
- Sites 403 不影响 GitHub Pages 发布或飞书推送。

### 10.2 日期筛选

- “全部日期”显示全部历史记录。
- 选择任一日期只显示该日记录。
- 日期与关键词、平台、结论组合筛选结果正确。
- React 页面与静态 GitHub Pages 行为一致。

### 10.3 月销售额

- 有完整输入时，美元区间计算符合公式与四舍五入规则。
- 缺少销量、价格或汇率任一关键输入时，不显示伪造金额。
- 页面显示单链接、周期、方法、可信度与来源。
- 旧简报可正常渲染且显示“暂无可靠估算”。
- 异常输入（负销量、上下限倒置、零汇率、非有限数）被校验拒绝。

### 10.4 测试

- 新增过滤函数测试、静态筛选脚本测试、汇率计算测试、缺失数据测试及渲染测试。
- 现有测试全部通过。
- 发布前对首页、历史页和至少一份简报执行本地及线上可访问性检查。

## 11. 实施边界

本次实施只负责页面、数据模型、估算规则、测试、发布流程和自动任务口径。第三方付费数据源接入将在用户取得合法账号或接口授权后单独实施；在此之前仅使用公开可核查来源和可审计替代数据。
