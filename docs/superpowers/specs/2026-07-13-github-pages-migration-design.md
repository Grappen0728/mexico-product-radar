# 墨西哥新品雷达 GitHub Pages 迁移设计

## 目标

将当前会被 `chatgpt.site` 安全层拦截的公开入口迁移到 GitHub Pages，继续满足以下要求：

- 无密码公开访问；
- 首页展示当天推荐；
- 可查看历史推荐和单条详情；
- 飞书卡片跳转到当天详情页；
- 每天 10:07 自动研究、发布、验证后再推送飞书；
- 任何密钥和本地配置不得进入公开仓库。

## 架构

创建公开仓库 `mexico-product-radar`，使用 GitHub Pages 托管纯静态页面。现有 JSON 推荐记录继续作为唯一内容来源，由本地构建脚本生成首页、历史页、趋势页和每个产品的独立详情页。

生产地址采用 GitHub Pages 自动分配的公开网址。原 `chatgpt.site` 保留为备用，但飞书和自动任务只使用新的 GitHub Pages 地址。

## 数据与发布流程

1. 每日任务联网核验一款合格产品，生成 `data/recommendations/YYYY-MM-DD-product-slug.json`。
2. 同步历史数据并生成静态网站。
3. 执行单元测试、类型检查、构建和链接检查。
4. 提交并推送公开仓库，由 GitHub Pages 发布。
5. 等待公开详情页返回成功，并确认包含当天产品名称。
6. 将已验证的精确详情页地址写入飞书卡片并推送。

飞书发送器继续执行“报告必须已经进入发布数据”的本地保护；自动任务额外执行线上详情页验证，避免消息早于页面上线。

## 页面与链接

静态网站保持现有深色仪表盘风格和内容结构：

- `/`：当天推荐和最近历史；
- `/archive/`：全部历史记录与筛选；
- `/trends/`：趋势概览；
- `/recommendations/<slug>/`：完整数据、推荐理由、价格、风险、图片、视频和来源。

所有站内链接使用仓库子路径兼容形式，确保 GitHub Pages 项目站点下的导航不会跳回域名根目录。

## 安全与公开范围

公开仓库只包含网站源代码、公开商品数据、公开来源链接和静态资源。以下内容必须保持忽略且禁止提交：

- 飞书 webhook；
- 飞书签名密钥；
- Sites 访问令牌；
- 本地环境文件和临时输出；
- 任何账号凭据。

发布前检查 Git 变更和仓库历史，发现疑似密钥时立即停止。

## 故障处理

- GitHub Pages 未部署成功：不发送飞书，保留上一个可用版本并报告失败原因。
- 当天详情页无法访问或内容不匹配：不发送飞书，重新构建或发布后再次验证。
- 当天没有证据充分的产品：不创建假记录；发送“今日无合格推荐”简报。
- GitHub 在特定网络下不可访问：保留飞书内完整摘要，并在后续接入自有域名作为第二公开入口。

## 验收标准

- 未登录浏览器可以打开新首页和当天详情页；
- 飞书最新卡片能打开精确详情页；
- 历史页可以看到已有推荐；
- 所有自动测试和静态链接检查通过；
- 公开仓库和构建产物不包含密钥；
- 每日自动任务严格遵循“发布成功 → 线上验证 → 飞书推送”。
