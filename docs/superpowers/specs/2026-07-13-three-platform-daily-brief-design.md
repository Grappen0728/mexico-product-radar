# 墨西哥三平台带电带芯片选品日报设计

## 目标

将“墨西哥新品雷达”从每日一款产品升级为每日三款产品：TikTok Shop Mexico、Amazon Mexico、Mercado Libre Mexico 各推荐一款最适合该平台测试的产品，并给出统一排名和唯一优先测试建议。

每天三款产品必须同时满足：

- 属于带电或用电产品；
- 含芯片或具备由芯片实现的核心电子功能；
- 三个平台的推荐产品互不相同；
- 公开证据足以支持平台适配、市场需求和价格判断；
- 不属于明显侵权、违法、医疗夸大、危险改装或高售后风险产品。

## 日报结构

一份日报包含三个平台板块和一个跨平台总结：

1. TikTok Shop Mexico 今日推荐；
2. Amazon Mexico 今日推荐；
3. Mercado Libre Mexico 今日推荐；
4. 今日三个产品 TOP 1–3 排名；
5. 如果只能测试一款产品时的优先选择和理由。

每个平台板块保存产品名称、品类、推荐理由、平台专项分析、星级评分、商业模型、风险、测试建议、公开来源、图片和视频素材。

## 平台专项分析

### TikTok Shop Mexico

重点分析：

- 短视频前三秒能否展示明显变化、反差或结果；
- 产品是否解决可视化强、描述成本低的用户痛点；
- 是否适合达人演示、开箱、对比或场景植入；
- 达人佣金和投流后是否仍有利润；
- 近 7–30 天是否出现内容、榜单、销量或广告增长信号。

必须输出三个具体短视频切入点，以及采购成本、建议售价、单件预估毛利、适合佣金和目标 ROI。

### Amazon Mexico

重点分析：

- 搜索需求及可公开核验的替代指标；
- 消费者购买原因和核心搜索意图；
- 头部 Listing 的评价、品牌和价格壁垒；
- 产品是否具备常青需求和合理生命周期；
- 可通过规格、套装、配件、说明书或售后形成的差异化。

必须输出建议价格、利润空间和 Listing 核心卖点。

### Mercado Libre Mexico

重点分析：

- 墨西哥本地消费者的实际需求和购买场景；
- 目标消费者画像及价格敏感度；
- 平台结果数量、热卖标签、评价和履约信号；
- 同类卖家竞争、价格带和物流适配；
- 站内广告、优惠券、套装和本地化标题的推广机会。

必须输出价格区间、主要卖点和推广方式。

## 评分口径

所有星级均采用 1–5 星，星级越高代表商业机会越好。

- 视频传播潜力：越高越容易通过短视频传播；
- 市场需求：越高代表需求信号越强；
- 竞争机会：越高代表竞争越低或差异化空间越大；
- 利润空间：越高代表扣除采购、运输、平台费用和佣金后的空间越充足。

不得把“竞争激烈”直接评为高星。页面和飞书统一使用“竞争机会”文案，避免误读。

跨平台排名采用 100 分制：

- 平台适配度：25 分；
- 市场需求：20 分；
- 内容、搜索或本地销售潜力：20 分；
- 利润空间：20 分；
- 竞争机会：10 分；
- 合规、物流和售后可控性：5 分。

最终排名必须同时给出总分和一句核心理由。TOP 1 即“只能测试一个产品”的默认选择；如果风险或供应链条件使 TOP 1 暂时不可执行，必须明确说明并选择下一名。

## 数据与证据

每款产品至少包含两个独立、可点击且与结论直接相关的来源，优先级如下：

1. 对应平台商品页、搜索页、榜单或官方趋势页；
2. Google Trends、TikTok Creative Center、YouTube 公开内容；
3. 品牌、制造商、说明书或认证资料；
4. 供应端公开价格和规格页面。

所有价格优先使用 MXN，注明查询日期。销量、搜索量、GMV、增长率或佣金无法公开核验时必须写“未公开”，并说明采用的替代信号。搜索摘要不能作为已验证事实。

每天先检查最近 30 天历史。禁止重复推荐同一产品；只有用户以后明确允许时才可调整此规则。

## 数据模型与历史

每天保存一个 `DailyPlatformBrief`：

- `date`：日报日期；
- `slug`：日报公开链接标识；
- `recommendations`：固定三项，分别对应 TikTok Shop Mexico、Amazon Mexico、Mercado Libre Mexico；
- `ranking`：三项产品排名、总分和理由；
- `priorityPick`：唯一优先测试产品和原因；
- `sourcesCheckedAt`：最终核验时间。

每个平台推荐对象包含产品基础信息、电子依据、平台专项分析、评分、商业模型、素材、来源、风险和测试建议。验证器必须保证平台不重复、产品 slug 不重复、三款均满足电子依据、来源数量达标且排名覆盖全部产品。

现有单产品历史作为“旧版记录”继续保留，不伪造为三平台日报。新版历史从第一份真实三平台日报开始。

## 网站

公开站点继续使用 GitHub Pages，无密码访问。

- 首页：展示当天三个平台推荐、TOP 1 和优先测试结论；
- 日报详情：`/briefs/YYYY-MM-DD/`，完整呈现用户指定的三平台模板；
- 产品详情：保留 `/recommendations/<slug>/`，展示证据、电子依据、供应链和素材；
- 历史页：按日期、平台、产品关键词和结论查询；
- 趋势页：统计平台、品类、评分和历史推荐分布。

移动端按 TikTok、Amazon、Mercado Libre 的顺序纵向排列，桌面端使用清晰的三平台分区，保持现有深色仪表盘风格。

## 飞书

每天只推送一张汇总卡，包含：

- 日报日期；
- 三个平台的产品名称、建议售价和核心分数；
- TOP 1 产品与优先测试理由；
- 一个“查看完整三平台日报”按钮。

飞书按钮必须指向已验证的 GitHub Pages 日报详情页。网站部署成功且日报页面包含当天三个产品名称后，才允许推送。

## 自动任务

执行时间保持每天北京时间 10:07，顺序固定：

1. 联网研究三个平台；
2. 筛选三款互不相同且带电含芯片的产品；
3. 生成并验证日报 JSON；
4. 同步网站历史并运行全部测试和构建；
5. 检查密钥和无关改动后提交；
6. 推送 GitHub 并等待 Pages 部署成功；
7. 验证当天日报页和三个产品详情页；
8. 推送一张飞书汇总卡；
9. 返回三个产品、排名、网站链接和执行状态。

## 异常处理

- 某个平台没有合格产品：该板块明确写“今日无合格推荐”，列出最接近候选和淘汰原因；不得用低证据产品凑数。
- 三个平台最终候选重复：保留平台适配度最高的一处，其他平台重新选品。
- 数据无法核验：降低需求或趋势评分，并明确标注缺失数据。
- 测试、构建、部署或线上验证失败：不发送飞书，先诊断并修复。
- 某产品运输、认证或售后风险过高：不得进入 TOP 1；必要时直接淘汰。

## 验收标准

- 每日日报最多三个推荐，正常情况下三个产品互不相同；
- 三款推荐均提供带电与含芯片依据；
- 三个平台板块完整覆盖用户指定字段；
- 竞争机会星级方向统一且无歧义；
- TOP 1–3 覆盖三个产品并给出唯一优先测试选择；
- 公开网页无需登录且可查看日报和历史；
- 飞书只在网页验证成功后发送；
- 仓库和构建产物不包含任何密钥。
